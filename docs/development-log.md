### 四、遇到的问题与修复

#### 问题 1：bcrypt / passlib 兼容性错误

**现象：**  
注册时出现误导性报错：
```text
ValueError: password cannot be longer than 72 bytes
```

**实际原因：**  
`passlib` 与较新版本 `bcrypt` 之间存在兼容性问题。

**修复方式：**  
将 `bcrypt` 降级到 `4.3.0`。

**结果：**  
注册与登录恢复正常。

---

#### 问题 2：MySQL 官方 APT 仓库签名过期

**现象：**  
执行 `apt update` 时出现 MySQL 仓库 GPG 签名错误。

**当前状态：**
- 不影响当前开发
- 后续若需要维护系统包，应再修复该问题

**处理：**

- 聊天业务 WebSocket 走 Vite 代理链路时稳定性不足

**处理：**
- 将业务 API 与业务 WebSocket 代理职责转移到 Nginx
- Vite 只负责前端页面与 HMR
- 调整 Nginx `/ws/` 反向代理配置
- 调整 `vite.config.js` 中 HMR 配置

**结果：**
- 聊天业务 WebSocket 链路更清晰
- 开发环境职责分层更合理
- 终端错误显著减少，整体稳定性提升

### 当前阶段遇到的问题与修复记录

#### 问题 1：上传路径公网访问返回 404
**现象：**
- 浏览器请求 `POST /uploads` 返回 `404 Not Found`

**实际原因：**
- Nginx 站点配置未正确转发 `/uploads`
- 仅有聊天 API 与 WebSocket 路径，缺少上传路径转发

**处理：**
- 补充 `location = /uploads`
- 补充 `location /uploads/`
- 重载 Nginx

**结果：**
- 公网上传路径恢复正常

---

#### 问题 2：上传文件返回 413 Request Entity Too Large
**现象：**
- 文件上传被 Nginx 拦截
- 返回 `413 Request Entity Too Large`

**实际原因：**
- Nginx 默认请求体大小限制过小

**处理：**
- 增加：
  ```nginx
  client_max_body_size 50M;
  ```

**结果：**
- 上传大小限制与项目规划一致
- 文件上传恢复正常

---

#### 问题 3：WebSocket 断开时后端刷红色异常日志
**现象：**
- 切房间、刷新页面或上传过程中偶发：
  - `WebSocketDisconnect`
  - `RuntimeError: WebSocket is not connected`

**实际原因：**
- 客户端已断开，但服务端协程仍在 `send_json` 或 `receive_json`

**处理：**
- 在 `ws.py` 中增加断连保护
- 对 `receive_json()` 增加异常处理
- 对发送动作增加安全封装
- 统一在 `finally` 中清理连接

**结果：**
- 连接关闭时不再反复刷出大段异常栈
- 开发日志明显更干净
